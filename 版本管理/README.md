## 📊 文档命名规范（V2.5+）

所有版本统一使用以下12个标准文档：

| 序号 | 文档 | 用途 | 编写时机 |
|:---:|------|------|----------|
| 00 | 文档索引.md | 导航和总览 | 版本启动时 |
| 01 | 规划.md | 版本目标和计划 | 版本启动时 |
| 02 | 需求.md | 功能需求详细描述 | 需求分析阶段 |
| 03 | 架构.md | 系统设计 | 设计阶段 |
| 04 | 技术.md | 技术实现细节 | 设计阶段 |
| 05 | 更新介绍.md | 对外发布说明 | 发布前 |
| 06 | 测试计划.md | 测试策略和用例 | 测试阶段前 |
| 07 | 测试报告.md | 测试结果 | 测试完成后 |
| 08 | 问题清单.md | Bug跟踪 | 全程维护 |
| 09 | 迭代日志.md | 开发过程记录 | 全程维护 |
| 10 | 发布说明.md | 部署和回滚指南 | 发布前 |
| 11 | 状态看板.md | 实时进度跟踪 | 全程维护 |

**命名规则**:
- 必须使用中文命名
- 必须带序号前缀（00-11）
- 使用.md后缀
