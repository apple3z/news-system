# 📝 文档撰写规范

---

## 一、文档分类

### 1.1 管理规范文档（总规范）
| 文档名称 | 说明 |
|----------|------|
| 研发管理规范 | 总体研发流程规范 |
| 文档撰写规范 | 文档格式标准 |

### 1.2 版本管理文档（每个版本独立）
| 类型 | 说明 |
|------|------|
| 更新介绍 | 版本目标、更新内容、开发状态 |
| 功能介绍 | 功能清单、技术实现 |
| BUG修复 | 问题修复、测试报告 |
| 源代码 | Release版本代码位置 |
| 测试用例 | 测试用例库 |
| 测试报告 | 测试执行结果 |
| 运维报告 | 发布记录、运维手册 |

### 1.3 研发项目管理文档
| 类型 | 说明 | 模板 |
|------|------|------|
| 研发计划 | 项目规划、里程碑 | DEV_PLAN_v{X}.md |
| 需求文档 | 产品需求定义 | PRD_v{X}.md |
| 架构设计 | 系统架构设计 | ARCH_v{X}.md |
| 代码设计 | 详细设计 | CODING_v{X}.md |
| 数据库设计 | 数据模型设计 | DB_v{X}.md |
| 测试计划 | 测试计划 | TEST_PLAN_v{X}.md |
| 测试用例 | 测试用例库 | TEST_CASE_v{X}.md |

## 二、命名规范

### 2.1 总规范文档命名
```
研发管理规范.md
文档撰写规范.md
```

### 2.2 版本文档命名
```
{类型}_v{版本}.md

示例：
PRD_v2.6.md        - v2.6版本需求文档
ARCH_v2.6.md       - v2.6版本架构文档
DEV_PLAN_v2.6.md   - v2.6版本研发计划
```

### 2.3 版本号规则
- 开发版本：vX.Y
- 开发中：vX.Y-developing
- 发布版本：vX.Y-release-YYYYMMDD
