## 一、架构原则

### 1.1 三层架构

本系统采用清晰的三层架构，确保职责分离：

```
Routes (API)  →  Services (业务逻辑)  →  DAL (数据访问)  →  SQLite/文件系统
   |                    |                       |
  Flask              纯函数                   sqlite3
  request/           无HTTP依赖              直接SQL
  response
```

### 1.2 各层职责

| 层级 | 职责 | 约束 |
|------|------|------|
| **Routes** | HTTP请求解析、参数校验、响应封装 | 禁止业务逻辑 |
| **Services** | 业务逻辑处理、数据转换 | 纯函数，不引用Flask对象 |
| **DAL** | SQL操作、文件I/O | 返回dict/list，不返回Flask对象 |

## 二、Route 层规范

### 2.1 代码结构

```python
@news_crawler_bp.route('/api/news/search')
def api_news_search():
    # 1. 参数获取
    keyword = request.args.get('keyword', '')
    page = int(request.args.get('page', 1))
    
    # 2. 参数校验
    if page < 1:
        return jsonify({'error': 'Invalid page'}), 400
    
    # 3. 调用Service
    result = news_service.search_news(keyword, page)
    
    # 4. 响应封装
    return jsonify(result)
```

### 2.2 规范要求

- 使用蓝图(Blueprint)注册路由
- 参数获取使用 `request.args` / `request.json`
- 错误返回使用 `jsonify({'error': 'message'}), status_code`
- 成功返回使用 `jsonify(data)`

├── skills_api.py      # /api/skills/*

## 三、Service 层规范

### 3.1 代码结构

```python
def search_news(keyword, page, per_page=20):
    """搜索新闻（纯函数，可独立测试）"""
    # 1. 数据查询
    news_list = news_dal.get_news_by_keyword(keyword, page, per_page)
    
    # 2. 数据转换
    result = [
        {
            'id': n['id'],
            'title': n['title'],
            'source': n['source']
        }
        for n in news_list
    ]
    
    # 3. 返回结果
    return {
        'data': result,
        'page': page,
        'total': len(news_list)
    }
```

### 3.2 规范要求

- **纯函数**：不引用 `request`, `response`, `session`
- **可测试**：不依赖Flask上下文，可直接调用
- **单一职责**：一个函数只做一件事
- **返回值**：返回 `dict` 或 `list`，不返回Flask对象

### 3.3 服务文件组织

```
modules/news_crawler/services/
├── news_service.py
├── skills_service.py
├── subscribe_service.py
└── crawler_service.py
```

## 四、DAL 层规范

### 4.1 代码结构

```python
def get_news_by_keyword(keyword, page, per_page):
    """获取新闻列表"""
    conn = get_db_connection('news.db')
    try:
        offset = (page - 1) * per_page
        cursor = conn.execute(
            'SELECT * FROM news WHERE title LIKE ? LIMIT ? OFFSET ?',
            (f'%{keyword}%', per_page, offset)
        )
        rows = cursor.fetchall()
        return [dict(row) for row in rows]
    finally:
        conn.close()
```

### 4.2 规范要求

- 使用参数化查询，防止SQL注入
- 使用 `try/finally` 确保连接关闭
- 返回 `dict` 或 `list`，不返回游标对象
- 复杂SQL添加注释说明

### 4.3 DAL文件组织

```
modules/news_crawler/dal/
├── news_dal.py
├── skills_dal.py
└── subscribe_dal.py
```

## 五、模块间通信

### 5.1 代理模式

使用 `proxy/registry.py` 实现跨模块通信：

```python
# 模块A：注册服务
from proxy.registry import register

def search_news(keywords):
    # 业务逻辑
    return results

register('news.search', search_news)

# 模块B：调用服务
from proxy import call

results = call('news.search', keywords=['AI', 'ML'])
```

### 5.2 规范要求

- **禁止跨模块直接import**：不允许 `from modules.news_crawler import ...`
- **使用register()注册**：在模块 `__init__.py` 中注册服务
- **使用call()调用**：通过服务名调用，不直接依赖

## 七、代码示例

### 7.1 新增一个API接口

**步骤1**：在DAL层添加数据访问方法
```python
# modules/news_crawler/dal/news_dal.py
def create_news(title, content):
    conn = get_db_connection('news.db')
    try:
        cursor = conn.execute(
            'INSERT INTO news (title, content) VALUES (?, ?)',
            (title, content)
        )
        conn.commit()
        return {'id': cursor.lastrowid, 'title': title}
    finally:
        conn.close()
```

**步骤2**：在Service层添加业务逻辑
```python
# modules/news_crawler/services/news_service.py
def create_news(title, content):
    if not title:
        raise ValueError('标题不能为空')
    return news_dal.create_news(title, content)
```

**步骤3**：在Route层添加API接口
```python
# modules/news_crawler/routes/news_api.py
@news_crawler_bp.route('/api/news', methods=['POST'])
def api_create_news():
    data = request.json
    try:
        result = news_service.create_news(data['title'], data.get('content', ''))
        return jsonify({'success': True, 'data': result})
    except ValueError as e:
        return jsonify({'success': False, 'error': str(e)}), 400
```
