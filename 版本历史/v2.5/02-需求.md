## 二、功能性需求

### 2.1 统一文档 API（新增核心需求）

#### 【需求编号】REQ-API-001
**需求名称**: 统一文档写入 API

**需求描述**: 
开发统一的文档 API，AI 通过 API 写入文档（不直接操作磁盘），API 自动调用版本管理函数，实现真正的版本控制。

**API 设计**:
```python
POST /api/v2/doc/save
{
    "path": "版本历史/v2.5/02-需求.md",
    "content": "# 内容...",
    "author": "瑞贝卡"
}
```

**验收标准**:
- [ ] AI 通过 API 写入文档，不直接操作磁盘
- [ ] 每次保存自动递增版本号（修订号 +1）
- [ ] 保存后立即写入物理文件
- [ ] Web 端刷新后立即显示新内容
- [ ] 版本历史记录完整（版本号、修改时间、修改人）

**优先级**: P0
