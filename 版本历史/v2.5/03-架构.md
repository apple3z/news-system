## 一、系统架构

### 1.1 整体架构

```
┌─────────────┐         ┌──────────────┐         ┌─────────────┐
│   Web 端     │ ──────► │  Flask App   │ ──────► │  物理文件   │
│  (浏览器)    │  HTTP   │   (app.py)   │  文件系统 │ (版本历史/) │
└─────────────┘         └──────────────┘         └─────────────┘
                              │
                              ▼
                       ┌──────────────┐
                       │ .version.json│
                       │ (版本信息)    │
                       └──────────────┘
```

### 1.2 模块划分

| 模块 | 职责 | 文件 |
|------|------|------|
| **Web 前端** | 文档展示、编辑、保存 | `templates/sys_v2.html`, `static/sys_v2.js` |
| **API 层** | 统一文档 API | `app.py` (新增路由) |
| **版本管理** | 自动版本递增、持久化 | `app.py` (版本管理函数) |
| **文件系统** | 物理文件读写 | Python `os`, `json` 模块 |

## 二、API 架构

### 2.1 统一文档 API

```python
# 新增路由
POST /api/v2/doc/save      # 保存文档（自动版本管理）
GET  /api/v2/doc/read      # 读取文档
GET  /api/v2/doc/history   # 获取版本历史
GET  /api/v2/doc/list      # 列出目录
```

### 2.2 版本管理流程

```
AI/Web 保存请求
    │
    ▼
验证路径安全
    │
    ▼
获取当前版本
    │
    ▼
递增版本号 (修订号 +1)
    │
    ▼
写入物理文件
    │
    ▼
更新 .version.json
    │
    ▼
清除缓存
    │
    ▼
返回成功响应
```

## 三、数据设计

### 3.1 版本信息存储 (.version.json)

**位置**: 每个版本目录下（如 `版本历史/v2.5/.version.json`）

**结构**:
```json
{
    "02-需求.md": {
        "version": "1.0.5",
        "mtime": "2026-03-09 18:30",
        "author": "瑞贝卡",
        "history": [
            {"version": "1.0.5", "mtime": "2026-03-09 18:30", "author": "瑞贝卡"},
            {"version": "1.0.4", "mtime": "2026-03-09 18:00", "author": "张振中"}
        ]
    },
    "01-规划.md": {
        ...
    }
}
```

### 3.2 版本号规则

```
格式：主版本。次版本。修订号
示例：1.0.0

递增规则:
- 每次保存：修订号 +1 (1.0.0 → 1.0.1)
- 重大变更：次版本 +1 (1.0.5 → 1.1.0)
- 架构重构：主版本 +1 (1.9.0 → 2.0.0)
```

## 五、安全设计

### 5.1 路径安全验证

```python
def is_safe_path(path):
    """验证路径安全，禁止路径遍历攻击"""
    allowed_prefixes = ['版本历史/', '文档中心/']
    return any(path.startswith(prefix) for prefix in allowed_prefixes)
```

### 5.2 文件类型限制

- 只允许 `.md` 文件
- 禁止访问 `.py`, `.json`（除 `.version.json`）

## 六、性能设计

### 6.1 缓存策略

- **读取**: 每次请求重新扫描目录（数据量小）
- **保存**: 立即写入磁盘，清除相关缓存
- **版本信息**: 内存缓存 + 文件持久化

### 6.2 性能指标

| 指标 | 目标 |
|------|------|
| API 响应时间 | < 500ms |
| 文档保存时间 | < 1s |
| Web 端刷新时间 | < 2s |
