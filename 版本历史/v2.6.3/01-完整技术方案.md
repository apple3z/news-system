## 1. 版本目标

### 1.1 核心目标

将爬虫功能从 `news_crawler` 模块中**完全剥离**，作为独立的 `crawler_engine` 包，实现：

- **独立运行**：爬虫引擎可脱离 Flask 主应用独立运行
- **统一调度**：APScheduler 替代手动触发，支持定时/手动双模式
- **数据去重**：URL + SimHash 双重去重，彻底解决重复入库
- **全文提取**：Newspaper4k 替代 BeautifulSoup 纯文本，提取结构化文章
- **实时监控**：独立的爬虫管理面板，实时展示任务状态和采集进度
- **yt-dlp 视频**：支持 YouTube/Bilibili 视频元数据抓取

### 1.2 与现有系统的关系

```
v2.6.2 (当前)                        v2.6.3 (目标)
┌──────────────────┐                 ┌──────────────────┐
│   news_crawler   │                 │   news_crawler   │  ← 只保留业务层
│  ├ crawlers/     │                 │  ├ services/     │    (Service/DAL/Route)
│  ├ dal/          │    重构拆分→     │  ├ dal/          │
│  ├ services/     │                 │  └ routes/       │
│  └ routes/       │                 ├──────────────────┤
│                  │                 │  crawler_engine  │  ← 新增独立包
│ subscribe_manager│                 │  ├ scheduler/    │    (调度+采集+处理)
│ fetch_news.py    │                 │  ├ fetchers/     │
│ fetch_skills.py  │                 │  ├ processors/   │
└──────────────────┘                 │  ├ storage/      │
                                     │  └ monitor/      │
                                     └──────────────────┘
```

### 1.3 解决的历史问题

| 编号 | 问题 | 方案 |
|:---:|------|------|
| W1 | subscription_history 无去重 | URL + SimHash 双重去重引擎 |
| W2 | 订阅源重复4份 | 唯一索引 + 清理脚本 |
| W3 | 指纹机制不可靠 | 逐条 link 对比，废弃标题拼接指纹 |
| W5 | 历史数据 link 全为 NULL | 数据迁移脚本清理 |
| W8 | 无自动调度 | APScheduler 定时任务 |
| W9 | fetch_news.py 两份不一致 | 统一到 crawler_engine 包 |
| W4 | 标题去重截断30字 | 改用 link 优先去重 |
| W10 | 错误处理缺陷 | try/finally + 超时重试 |

## 2. 产品交互方案

### 2.1 导航结构变更

**现有结构（v2.6.2）**：
```
后台管理
├── 数据采集（一级Tab）
│   ├── 数据采集工作台 (/sys/news)     ← 爬虫+新闻混在一起
│   ├── Skills管理 (/sys/skills)
│   └── 订阅管理 (/sys/subscribe)
├── 基础管理（一级Tab）
│   ├── 版本历史 (/sys/versions)
│   ├── 文档中心 (/sys/docs)
│   └── 账户管理 (/sys/accounts)
```

**新结构（v2.6.3）**：
```
后台管理
├── 爬虫中心（一级Tab）               ← 新增，独立爬虫管理
│   ├── 任务总览 (/sys/crawler)       ← 实时看板
│   ├── 采集源管理 (/sys/crawler/sources)
│   ├── 采集日志 (/sys/crawler/logs)
│   └── 调度配置 (/sys/crawler/schedule)
├── 内容管理（一级Tab）               ← 原"数据采集"改名
│   ├── 新闻管理 (/sys/news)          ← 纯内容管理，不含爬虫
│   ├── Skills管理 (/sys/skills)
│   └── 订阅内容 (/sys/subscribe)
├── 基础管理（一级Tab）
│   ├── 版本历史 (/sys/versions)
│   ├── 文档中心 (/sys/docs)
│   └── 账户管理 (/sys/accounts)
```

```python
# crawler_engine/fetchers/base.py

from abc import ABC, abstractmethod

class BaseFetcher(ABC):
    """采集器抽象基类"""

    def __init__(self, config=None):
        self.config = config or {}
        self.timeout = self.config.get('timeout', 15)
        self.retries = self.config.get('retries', 2)
        self.user_agent = self.config.get('user_agent', 'Mozilla/5.0 ...')

    @abstractmethod
    def fetch(self, source):
        """
        采集单个源

        Args:
            source: dict, 数据源信息 {id, name, url, type, config}

        Returns:
            list[dict]: 采集到的条目列表，每条包含:
                - title: str
                - link: str (唯一标识)
                - summary: str
                - content: str (可选，全文)
                - pub_date: str
                - source_name: str
                - source_type: str ('news'|'rss'|'video'|'skill')
                - extra: dict (可选，附加数据)
        """
        pass

    def fetch_with_retry(self, source):
        """带重试的采集"""
        for attempt in range(self.retries + 1):
            try:
                return self.fetch(source)
            except Exception as e:
                if attempt < self.retries:
                    time.sleep(2 ** attempt)  # 指数退避
                else:
                    raise
```

```python
# crawler_engine/fetchers/rss_fetcher.py

import feedparser

class RSSFetcher(BaseFetcher):
    """RSS/Atom 采集器"""

    def fetch(self, source):
        feed = feedparser.parse(source['url'], request_headers={
            'User-Agent': self.user_agent
        })

        if feed.bozo and not feed.entries:
            raise FetchError(f"Feed parse error: {feed.bozo_exception}")

        items = []
        for entry in feed.entries:
            items.append({
                'title': entry.get('title', '').strip(),
                'link': self._clean_link(entry.get('link', '')),
                'summary': self._extract_summary(entry),
                'content': self._extract_content(entry),
                'pub_date': entry.get('published', '') or entry.get('updated', ''),
                'source_name': source['name'],
                'source_type': 'rss',
            })
        return items

    def _clean_link(self, link):
        """清理URL参数（?f=rss 等追踪参数）"""
        if not link:
            return ''
        for suffix in ['?f=rss', '?f=feed', '?utm_source=', '&utm_']:
            idx = link.find(suffix)
            if idx > 0:
                link = link[:idx]
        return link

    def _extract_summary(self, entry):
        """提取摘要（优先 summary，fallback content）"""
        if hasattr(entry, 'summary') and entry.summary:
            return self._strip_html(entry.summary)[:500]
        if hasattr(entry, 'content') and entry.content:
            return self._strip_html(entry.content[0].value)[:500]
        return ''

    def _extract_content(self, entry):
        """提取完整内容"""
        if hasattr(entry, 'content') and entry.content:
            return entry.content[0].value
        return entry.get('summary', '')

    def _strip_html(self, html):
        import re
        return re.sub(r'<[^>]*>', '', html).strip()
```

#### 5.2.6 DedupEngine - 去重引擎

```python
# crawler_engine/processors/dedup.py

import hashlib

class DedupEngine:
    """双重去重引擎: URL精确去重 + SimHash近似去重"""

    def __init__(self, db_path=None):
        self.db_path = db_path

    def is_duplicate(self, item, source_type='news'):
        """
        检查条目是否重复

        Returns:
            bool: True = 重复，应跳过
        """
        link = item.get('link', '')

        # 第一层：URL 精确去重
        if link and self._url_exists(link, source_type):
            return True

        # 第二层：SimHash 近似去重（标题相似度）
        title = item.get('title', '')
        if title and self._title_similar(title, source_type):
            return True

        return False

    def _url_exists(self, link, source_type):
        """检查URL是否已存在"""
        conn = self._get_conn(source_type)
        try:
            if source_type == 'news':
                row = conn.execute(
                    "SELECT id FROM news WHERE link = ?", (link,)
                ).fetchone()
            elif source_type == 'rss':
                row = conn.execute(
                    "SELECT id FROM subscription_history WHERE link = ?", (link,)
                ).fetchone()
            elif source_type == 'video':
                row = conn.execute(
                    "SELECT id FROM video_meta WHERE link = ?", (link,)
                ).fetchone()
            else:
                return False
            return row is not None
        finally:
            conn.close()

    def _title_similar(self, title, source_type, threshold=3):
        """
        SimHash 近似去重

        计算标题的 SimHash 值，与已有条目对比。
        海明距离 <= threshold 视为重复。
        """
        new_hash = self._simhash(title)

        conn = self._get_conn(source_type)
        try:
            # 取最近7天的标题哈希对比（限制范围避免性能问题）
            table = self._get_table(source_type)
            rows = conn.execute(
                f"SELECT title_hash FROM {table} WHERE created_at > datetime('now', '-7 days')"
            ).fetchall()

            for row in rows:
                if row[0] and self._hamming_distance(new_hash, int(row[0])) <= threshold:
                    return True
            return False
        finally:
            conn.close()

    @staticmethod
    def _simhash(text, hashbits=64):
        """计算文本的 SimHash 值"""
        tokens = list(text)  # 字符级分词（中文友好）
        v = [0] * hashbits

        for token in tokens:
            h = int(hashlib.md5(token.encode('utf-8')).hexdigest(), 16)
            for i in range(hashbits):
                bitmask = 1 << i
                if h & bitmask:
                    v[i] += 1
                else:
                    v[i] -= 1

        fingerprint = 0
        for i in range(hashbits):
            if v[i] >= 0:
                fingerprint |= (1 << i)
        return fingerprint

    @staticmethod
    def _hamming_distance(a, b):
        """计算两个整数的海明距离"""
        x = a ^ b
        count = 0
        while x:
            count += 1
            x &= x - 1
        return count
```

```python
# crawler_engine/scheduler/manager.py

from apscheduler.schedulers.background import BackgroundScheduler
from apscheduler.triggers.interval import IntervalTrigger

class SchedulerManager:
    """APScheduler 调度管理器"""

    DEFAULT_INTERVALS = {
        'news': {'hours': 2},
        'rss': {'minutes': 30},
        'skills': {'hours': 12},
        'video': {'hours': 6},
    }

    def __init__(self):
        self.scheduler = BackgroundScheduler()
        self.enabled = True
        self._intervals = dict(self.DEFAULT_INTERVALS)

    def load_config(self, app_config):
        """从 Flask config 加载调度配置"""
        intervals = app_config.get('CRAWLER_INTERVALS', {})
        self._intervals.update(intervals)

    def start(self):
        """启动调度器，注册所有任务"""
        from .jobs import run_news_job, run_rss_job, run_skills_job, run_video_job

        job_map = {
            'news': run_news_job,
            'rss': run_rss_job,
            'skills': run_skills_job,
            'video': run_video_job,
        }

        for job_type, func in job_map.items():
            interval = self._intervals.get(job_type, {'hours': 6})
            self.scheduler.add_job(
                func=func,
                trigger=IntervalTrigger(**interval),
                id=f'crawl_{job_type}',
                name=f'{job_type} crawl',
                replace_existing=True,
                max_instances=1,  # 同类任务不并发
            )

        self.scheduler.start()

    def shutdown(self):
        self.scheduler.shutdown(wait=False)

    def run_now(self, job_type):
        """立即触发一次任务"""
        job = self.scheduler.get_job(f'crawl_{job_type}')
        if job:
            job.modify(next_run_time=datetime.now())
            return True
        return False

    def update_interval(self, job_type, **kwargs):
        """更新任务间隔"""
        self._intervals[job_type] = kwargs
        job = self.scheduler.get_job(f'crawl_{job_type}')
        if job:
            job.reschedule(trigger=IntervalTrigger(**kwargs))

    def toggle_job(self, job_type, enabled):
        """启用/停用单个任务"""
        job = self.scheduler.get_job(f'crawl_{job_type}')
        if job:
            if enabled:
                job.resume()
            else:
                job.pause()

    def get_status(self):
        """获取调度器状态"""
        jobs = []
        for job in self.scheduler.get_jobs():
            jobs.append({
                'id': job.id,
                'name': job.name,
                'next_run': str(job.next_run_time) if job.next_run_time else None,
                'interval': str(job.trigger),
                'paused': job.next_run_time is None,
            })
        return {
            'running': self.scheduler.running,
            'enabled': self.enabled,
            'jobs': jobs
        }
```

```python
# crawler_engine/monitor/tracker.py

import threading
from datetime import datetime

class TaskTracker:
    """任务状态跟踪器（内存中，用于实时UI展示）"""

    def __init__(self):
        self._tasks = {}       # {task_id: TaskInfo}
        self._stats = {}       # 统计缓存
        self._lock = threading.Lock()

    def start_task(self, task_id, task_type, total_sources=0):
        """标记任务开始"""
        with self._lock:
            self._tasks[task_id] = {
                'id': task_id,
                'type': task_type,
                'status': 'running',
                'start_time': datetime.now().isoformat(),
                'end_time': None,
                'total_sources': total_sources,
                'completed_sources': 0,
                'total_items': 0,
                'new_items': 0,
                'skipped_items': 0,
                'failed_items': 0,
                'progress': 0,
                'current_source': '',
                'errors': [],
            }

    def update_progress(self, task_id, source_name, result):
        """更新单源采集进度"""
        with self._lock:
            task = self._tasks.get(task_id)
            if not task:
                return
            task['completed_sources'] += 1
            task['total_items'] += result.get('total', 0)
            task['new_items'] += result.get('new', 0)
            task['skipped_items'] += result.get('skipped', 0)
            task['failed_items'] += result.get('failed', 0)
            task['current_source'] = source_name
            if task['total_sources'] > 0:
                task['progress'] = int(
                    task['completed_sources'] / task['total_sources'] * 100
                )

    def finish_task(self, task_id, status='success'):
        """标记任务完成"""
        with self._lock:
            task = self._tasks.get(task_id)
            if task:
                task['status'] = status
                task['end_time'] = datetime.now().isoformat()
                task['progress'] = 100

    def get_active_tasks(self):
        """获取正在运行的任务"""
        with self._lock:
            return [t for t in self._tasks.values() if t['status'] == 'running']

    def get_recent_tasks(self, limit=20):
        """获取最近的任务（包括已完成）"""
        with self._lock:
            tasks = sorted(self._tasks.values(),
                          key=lambda t: t['start_time'], reverse=True)
            return tasks[:limit]

    def get_stats(self):
        """获取今日统计"""
        today = datetime.now().strftime('%Y-%m-%d')
        with self._lock:
            today_tasks = [
                t for t in self._tasks.values()
                if t['start_time'].startswith(today)
            ]
            return {
                'today_total': sum(t['total_items'] for t in today_tasks),
                'today_new': sum(t['new_items'] for t in today_tasks),
                'today_tasks': len(today_tasks),
                'success_rate': self._calc_success_rate(today_tasks),
            }

    def _calc_success_rate(self, tasks):
        if not tasks:
            return 0
        success = sum(1 for t in tasks if t['status'] == 'success')
        return round(success / len(tasks) * 100, 1)
```

### 7.3 数据迁移脚本

```python
# scripts/migrate_v263.py

"""v2.6.3 数据迁移脚本"""

def migrate():
    """执行所有迁移"""

    # 1. 清理重复订阅源
    clean_duplicate_subscriptions()

    # 2. 清理旧的无效历史数据 (link 为 NULL)
    clean_null_link_history()

    # 3. 新增表和字段
    create_video_meta_table()
    create_crawler_schedule_table()
    alter_existing_tables()

    # 4. 为已有新闻计算 SimHash 指纹
    backfill_title_hashes()

    # 5. 添加唯一索引防止未来重复
    add_unique_constraints()

def clean_duplicate_subscriptions():
    """清理重复的订阅源"""
    conn = sqlite3.connect(SUBSCRIBE_DB)
    # 保留每个URL最早的一条
    conn.execute("""
        DELETE FROM subscription WHERE id NOT IN (
            SELECT MIN(id) FROM subscription GROUP BY url
        )
    """)
    # 添加唯一索引
    conn.execute("CREATE UNIQUE INDEX IF NOT EXISTS idx_sub_url ON subscription(url)")
    conn.commit()
    conn.close()

def clean_null_link_history():
    """清理 link 为空的旧数据"""
    conn = sqlite3.connect(SUBSCRIBE_DB)
    conn.execute("DELETE FROM subscription_history WHERE link IS NULL OR link = ''")
    conn.commit()
    conn.close()
```

## 9. 下一步规划

### v2.7.x — 高级采集能力（仅规划，不开发）

以下功能在 v2.6.3 的 `crawler_engine` 架构基础上，通过新增 Fetcher 插件即可扩展：

#### 9.1 Playwright JS 渲染采集

**目的**：采集需要 JavaScript 渲染的页面（SPA 站点、动态加载内容）

**实现方式**：
- 新增 `crawler_engine/fetchers/playwright_fetcher.py`
- 继承 `BaseFetcher`，使用 Playwright 无头浏览器
- 需要安装 Chromium (~150MB)

**适用场景**：
- 微博热搜、知乎热榜（反爬严格）
- 动态渲染的新闻站点（部分 React/Vue SPA）

**新增依赖**：`playwright>=1.40.0`

#### 9.2 Crawl4AI / Firecrawl 集成

**目的**：利用大模型理解页面结构，自动提取结构化数据

**实现方式**：
- 新增 `crawler_engine/fetchers/ai_fetcher.py`
- 支持两种后端：
  - Crawl4AI（开源免费，本地运行）
  - Firecrawl（SaaS，按量付费）

**适用场景**：
- 无 RSS 且页面结构复杂的站点
- 需要从非标准页面提取特定字段

**新增依赖**：`crawl4ai>=0.3.0`

#### 9.3 搜索引擎聚合

**目的**：从搜索引擎获取特定话题的最新报道

**实现方式**：
- 新增 `crawler_engine/fetchers/search_fetcher.py`
- 支持：
  - SerpAPI（Google/Bing/Baidu，付费 $50/月）
  - Serper.dev（Google，$4/1000次）
  - DuckDuckGo（免费，限速严格）

**适用场景**：
- 按关键词追踪特定话题
- 发现非订阅源的新闻报道

#### 9.4 社交媒体监控

**目的**：追踪 Twitter/微博 等平台的科技话题

**实现方式**：
- 新增 `crawler_engine/fetchers/social_fetcher.py`
- Twitter: `twscrape` 或官方 API v2
- 微博: Playwright 模拟 + Cookie

**适用场景**：
- 科技大V动态
- 突发科技事件追踪

**风险**：
- 平台政策限制，账号可能被封
- 需要维护 Cookie/Token

#### 9.5 v2.7.x 技术选型对比

| 能力 | 工具 | 费用 | 部署复杂度 | 优先级 |
|------|------|------|:----------:|:------:|
| JS 渲染 | Playwright | 免费 | 中（需Chromium） | 🟡 中 |
| AI 提取 | Crawl4AI | 免费 | 低 | 🟡 中 |
| AI 提取 | Firecrawl | $19/月起 | 低 | 🟢 低 |
| 搜索聚合 | SerpAPI | $50/月 | 低 | 🟡 中 |
| 搜索聚合 | Serper.dev | $4/1k次 | 低 | 🟡 中 |
| 社交媒体 | twscrape | 免费 | 高（账号维护） | 🔴 低 |

#### 9.6 架构兼容性

v2.6.3 的 `crawler_engine` 架构已预留扩展点：

```
v2.6.3 已实现:
  fetchers/
    ├── base.py           ← 基类
    ├── rss_fetcher.py    ← RSS
    ├── web_fetcher.py    ← 网页
    ├── video_fetcher.py  ← 视频
    └── skill_fetcher.py  ← Skills

v2.7.x 只需新增:
  fetchers/
    ├── playwright_fetcher.py   ← JS渲染
    ├── ai_fetcher.py           ← AI提取
    ├── search_fetcher.py       ← 搜索聚合
    └── social_fetcher.py       ← 社交媒体
```

每个新 Fetcher 只需：
1. 继承 `BaseFetcher`
2. 实现 `fetch(source)` 方法
3. 返回标准化的条目列表

调度器、去重引擎、存储层、监控层**无需任何改动**。
