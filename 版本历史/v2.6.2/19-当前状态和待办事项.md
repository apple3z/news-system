#### 3. Service 层实现

**文件**: `modules/news_crawler/services/subscribe_service.py`

**需要实现的 4 个函数**:

```python
def calculate_time_score(detected_at):
    """计算时间衰减分数
    公式：exp(-hours / 24)
    - 1 小时内：1.0
    - 24 小时后：0.5
    - 7 天后：0.1
    """
    import math
    hours = (now - detected_at).total_seconds() / 3600
    return math.exp(-hours / 24)

def get_digest_feeds(time_range, limit, source_weights):
    """获取摘要内容（智能排序）- 核心算法"""
    feeds = dal.get_digest_feeds_raw(time_range, limit * 2)
    for feed in feeds:
        weight = source_weights.get(feed['sub_id'], 3)
        time_score = calculate_time_score(feed['detected_at'])
        feed['sort_score'] = weight * time_score
    feeds.sort(key=lambda x: x['sort_score'], reverse=True)
    return feeds[:limit]

def get_source_weights():
    """获取源权重配置"""
    return dal.get_source_weights()

def set_source_weight(source_id, weight):
    """设置源权重"""
    dal.set_source_weight(source_id, weight)
```

## 🔧 当前问题

### 权重评分功能问题

**用户反馈**: "依然没有看到权重评分"

**现状**:
- ✅ 前端界面完整（星星显示、点击设置）
- ✅ 前端排序正常（localStorage 版本）
- ❌ 后端 API 未实现
- ❌ 权重未持久化（刷新页面后丢失）

**原因分析**:
1. 权重只保存在 localStorage，未保存到数据库
2. 前端排序是客户端计算，非服务端
3. 刷新页面后 localStorage 可能丢失
4. 用户可能没有切换到"按源浏览"视图查看星星设置

**临时解决方案**（当前可用）:
1. 访问 `/subscribe-digest`
2. 切换到"按源浏览"视图
3. 点击星星设置优先级
4. 返回"今日摘要"查看排序变化
5. **注意**: 刷新页面后权重会丢失

**永久解决方案**（需要后端实现）:
1. ✅ 数据库迁移
2. ⏳ DAL 层实现
3. ⏳ Service 层实现
4. ⏳ Route 层实现
5. ⏳ 前端改造（调用后端 API）
