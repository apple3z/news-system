## 1. 方案目标

将现有的"能用就行"爬虫升级为**工业级新闻采集系统**，对标 Google News / 今日头条 的采集架构思路，结合项目实际规模（单机 Flask + SQLite）做适度裁剪。

**核心目标**：
- 数据不重复、不丢失
- 自动化定时采集
- 可扩展新数据源
- 采集质量可监控

## 2. 业界最优方案调研

### 2.1 Google News 级架构（参考标杆）

```
Schedule → Fetch → Buffer → Process → Store → Serve
   调度      抓取     缓冲     处理      存储     服务
```

核心设计原则：
- **管道式架构**：每个阶段独立、可扩展
- **异步解耦**：抓取和处理通过消息队列解耦
- **多级去重**：URL精确去重 + 内容近似去重（SimHash/MinHash）
- **智能调度**：根据源的更新频率动态调整采集间隔
- **礼貌爬取**：遵守 robots.txt、限速、UA 轮转

> 参考：[Google News System Design](https://www.systemdesignhandbook.com/guides/google-news-system-design/) | [Scalable News Aggregator](https://medium.com/@123punit.sharma/designing-a-scalable-news-aggregator-system-a-step-by-step-technical-breakdown-700ddf8cb289)

### 2.2 Python 爬虫工具链对比

| 工具 | 定位 | 易用性 | 扩展性 | 提取精度 | 适合场景 |
|------|------|:------:|:------:|:--------:|----------|
| **feedparser** | RSS/Atom 解析 | ⭐⭐⭐⭐⭐ | ⭐⭐ | RSS专用 | RSS源解析（已在用） |
| **Newspaper4k** | 文章提取 | ⭐⭐⭐⭐ | ⭐⭐⭐ | F1=0.949 | 从网页提取正文/标题/图片 |
| **Fundus** | 高精度新闻提取 | ⭐⭐⭐⭐ | ⭐⭐ | F1=0.977 | 已知出版商的高质量提取 |
| **Scrapy** | 大规模爬虫框架 | ⭐⭐ | ⭐⭐⭐⭐⭐ | 需手写 | 大规模、复杂采集 |
| **Trafilatura** | 网页正文提取 | ⭐⭐⭐⭐ | ⭐⭐⭐ | F1=0.958 | 通用网页正文提取 |

> 参考：[Open-Source News Crawlers Comparison](https://github.com/free-news-api/news-crawlers) | [Fundus 论文](https://arxiv.org/abs/2403.15279) | [Python Web Scraping Libraries](https://www.scrapingbee.com/blog/best-python-web-scraping-libraries/)

**我们项目的最优组合**：

```
feedparser（RSS解析）+ Newspaper4k（正文提取）+ APScheduler（定时调度）
```

理由：
- feedparser 已在用，RSS 解析够用
- Newspaper4k 解决"正文提取质量差"的问题（自动提取标题、正文、图片、作者、发布时间）
- 不需要 Scrapy 的重量级框架（我们数据源 < 50 个）
- APScheduler 与 Flask 深度集成，零额外基础设施

### 2.3 去重算法对比

| 算法 | 原理 | 精度 | 内存 | 适合场景 |
|------|------|:----:|:----:|----------|
| **URL精确匹配** | link 字段 UNIQUE 约束 | 100%（相同URL） | 极低 | 基础层，必须有 |
| **SimHash** | 文本指纹，64位哈希 | 高 | 极低（每文档8字节） | 转载/小修改检测 |
| **MinHash + LSH** | 集合相似度，Jaccard | 高 | 较高 | 大规模模糊去重 |
| **语义向量** | Sentence Transformer | 最高 | 最高 | 洗稿/改写检测 |

> 参考：[Near-Duplicate Detection (NAACL 2025)](https://aclanthology.org/2025.naacl-industry.73.pdf) | [text-dedup 工具库](https://github.com/ChenghaoMou/text-dedup)

**我们项目的选择**：**URL精确匹配 + SimHash 近似去重**

理由：
- URL 去重解决 90% 的重复（同一文章同一链接）
- SimHash 解决转载场景（不同源转载同一篇文章，URL不同但内容相似）
- MinHash/语义向量对我们的规模来说 overkill

### 4.1 调度器 scheduler.py

```python
from apscheduler.schedulers.background import BackgroundScheduler
from apscheduler.jobstores.sqlalchemy import SQLAlchemyJobStore

class CrawlScheduler:
    """采集调度器"""

    def __init__(self, app=None):
        self.scheduler = BackgroundScheduler(
            jobstores={
                'default': SQLAlchemyJobStore(url='sqlite:///data/scheduler.db')
            },
            job_defaults={
                'max_instances': 1,           # 同一任务不并发
                'misfire_grace_time': 300,     # 错过5分钟内补执行
                'coalesce': True,              # 错过多次只补一次
            }
        )

    def init_app(self, app):
        """Flask 应用初始化时调用"""
        # RSS订阅采集：每30分钟
        self.scheduler.add_job(
            func=self._run_rss_crawl,
            trigger='interval', minutes=30,
            id='rss_crawl', replace_existing=True
        )
        # 新闻采集：每2小时
        self.scheduler.add_job(
            func=self._run_news_crawl,
            trigger='interval', hours=2,
            id='news_crawl', replace_existing=True
        )
        # Skills采集：每6小时
        self.scheduler.add_job(
            func=self._run_skills_crawl,
            trigger='interval', hours=6,
            id='skills_crawl', replace_existing=True
        )
        self.scheduler.start()

    def _run_rss_crawl(self):
        """执行RSS采集"""
        from .fetcher_rss import RSSFetcher
        fetcher = RSSFetcher()
        result = fetcher.fetch_all()
        self._log_result('rss', result)

    def _run_news_crawl(self):
        """执行新闻采集"""
        from .fetcher_web import WebFetcher
        fetcher = WebFetcher()
        result = fetcher.fetch_all()
        self._log_result('news', result)

    # 管理API（供后台管理页面调用）
    def get_jobs(self):
        """获取所有调度任务状态"""
        return [{
            'id': job.id,
            'next_run': str(job.next_run_time),
            'trigger': str(job.trigger),
        } for job in self.scheduler.get_jobs()]

    def trigger_now(self, job_id):
        """立即触发某个任务"""
        job = self.scheduler.get_job(job_id)
        if job:
            job.modify(next_run_time=datetime.now())

    def update_interval(self, job_id, minutes):
        """修改采集间隔"""
        self.scheduler.reschedule_job(
            job_id, trigger='interval', minutes=minutes
        )
```

### 4.2 RSS采集器 fetcher_rss.py

```python
import feedparser
from .dedup import DedupEngine
from .processor import ContentProcessor

class RSSFetcher:
    """RSS 采集器 - 使用 feedparser 解析"""

    def fetch_all(self):
        """采集所有活跃的RSS订阅源"""
        sources = self._get_active_sources()
        result = {'total': 0, 'new': 0, 'skipped': 0, 'errors': 0}

        for source in sources:
            try:
                items = self._fetch_one(source)
                result['total'] += len(items)
            except Exception as e:
                result['errors'] += 1
                logger.error(f"RSS采集失败: {source['name']} - {e}")

        return result

    def _fetch_one(self, source):
        """采集单个RSS源"""
        feed = feedparser.parse(source['url'])

        if feed.bozo and not feed.entries:
            raise Exception(f"Feed parse error: {feed.bozo_exception}")

        items = []
        for entry in feed.entries:
            item = {
                'title': entry.get('title', '').strip(),
                'link': self._clean_link(entry.get('link', '')),
                'summary': self._extract_summary(entry),
                'pub_date': entry.get('published', '') or entry.get('updated', ''),
                'source_name': source['name'],
                'source_id': source['id'],
            }

            # 第一层去重：URL精确匹配
            if DedupEngine.exists_by_link(item['link'], source['id']):
                continue

            items.append(item)

        # 批量入库
        ContentProcessor.save_batch(items, table='subscription_history')
        return items

    def _clean_link(self, link):
        """清理链接：移除追踪参数"""
        if not link:
            return ''
        from urllib.parse import urlparse, urlunparse, parse_qs, urlencode
        parsed = urlparse(link)
        # 移除常见追踪参数
        params = parse_qs(parsed.query)
        clean_params = {k: v for k, v in params.items()
                        if k not in ['f', 'utm_source', 'utm_medium', 'utm_campaign']}
        clean_query = urlencode(clean_params, doseq=True)
        return urlunparse(parsed._replace(query=clean_query))

    def _extract_summary(self, entry):
        """提取摘要：优先 summary，fallback content"""
        if hasattr(entry, 'summary') and entry.summary:
            return entry.summary[:500]
        if hasattr(entry, 'content') and entry.content:
            return entry.content[0].value[:500]
        return ''
```

### 4.3 网页采集器 fetcher_web.py（引入 Newspaper4k）

```python
from newspaper import Article, Config

class WebFetcher:
    """网页新闻采集器 - 使用 Newspaper4k 提取正文"""

    def __init__(self):
        self.config = Config()
        self.config.browser_user_agent = 'Mozilla/5.0 ...'
        self.config.request_timeout = 15
        self.config.memoize_articles = False

    def extract_article(self, url):
        """
        从URL提取完整文章信息
        返回: {title, content, summary, image, authors, pub_date}
        """
        article = Article(url, config=self.config, language='zh')
        article.download()
        article.parse()
        article.nlp()  # 自动生成摘要和关键词

        return {
            'title': article.title,
            'content': article.text,              # 干净的正文
            'summary': article.summary,           # NLP自动摘要
            'image': article.top_image,           # 主图URL
            'authors': ', '.join(article.authors),
            'keywords': article.keywords,         # NLP关键词
            'pub_date': str(article.publish_date) if article.publish_date else '',
        }

    def fetch_from_rss_list(self, sources):
        """
        先用 feedparser 获取文章列表
        再用 Newspaper4k 提取每篇正文
        """
        all_items = []
        for source in sources:
            feed = feedparser.parse(source['url'])
            for entry in feed.entries[:20]:  # 每源最多20条
                link = entry.get('link', '')
                if not link or DedupEngine.exists_by_link(link):
                    continue

                try:
                    detail = self.extract_article(link)
                    detail['link'] = link
                    detail['source_name'] = source['name']
                    all_items.append(detail)
                except Exception as e:
                    logger.warning(f"文章提取失败: {link} - {e}")

        return all_items
```

| content | 正文内容 | `get_text()` 纯文本 | 智能识别正文区域 |
| summary | 文章摘要 | 截取前500字 | NLP自动生成 |
| image | 主图 | 手写 CSS 选择器 | 自动识别 |

| pub_date | 发布时间 | 手写正则 | 自动识别 |
| keywords | 关键词 | 无 | NLP自动提取 |

### 4.4 去重引擎 dedup.py

```python
import hashlib

class SimHash:
    """
    SimHash 近似去重
    Google 用于网页去重的经典算法，64位指纹
    海明距离 ≤ 3 判定为近似重复
    """

    @staticmethod
    def compute(text, hashbits=64):
        """计算文本的 SimHash 值"""
        tokens = text.split()
        v = [0] * hashbits

        for token in tokens:
            token_hash = int(hashlib.md5(token.encode('utf-8')).hexdigest(), 16)
            for i in range(hashbits):
                bitmask = 1 << i
                if token_hash & bitmask:
                    v[i] += 1
                else:
                    v[i] -= 1

        fingerprint = 0
        for i in range(hashbits):
            if v[i] >= 0:
                fingerprint |= (1 << i)
        return fingerprint

    @staticmethod
    def hamming_distance(hash1, hash2):
        """计算海明距离"""
        x = hash1 ^ hash2
        count = 0
        while x:
            count += 1
            x &= x - 1
        return count

    @staticmethod
    def is_near_duplicate(hash1, hash2, threshold=3):
        """海明距离 ≤ 3 判定为近似重复"""
        return SimHash.hamming_distance(hash1, hash2) <= threshold


class DedupEngine:
    """
    三级去重引擎
    Level 1: URL 精确匹配（最快，解决90%重复）
    Level 2: 标题精确匹配（处理不同URL同一文章）
    Level 3: SimHash 近似匹配（处理转载/小修改）
    """

    @staticmethod
    def exists_by_link(link, source_id=None):
        """Level 1: URL 精确去重"""
        if not link:
            return False
        conn = _get_conn()
        try:
            row = conn.execute(
                "SELECT id FROM subscription_history WHERE link = ?", (link,)
            ).fetchone()
            return row is not None
        finally:
            conn.close()

    @staticmethod
    def exists_by_title(title, source_id=None):
        """Level 2: 标题精确去重"""
        if not title:
            return False
        conn = _get_conn()
        try:
            row = conn.execute(
                "SELECT id FROM subscription_history WHERE title = ? AND sub_id = ?",
                (title, source_id)
            ).fetchone()
            return row is not None
        finally:
            conn.close()

    @staticmethod
    def is_content_duplicate(content):
        """Level 3: SimHash 内容近似去重"""
        if not content or len(content) < 100:
            return False

        new_hash = SimHash.compute(content)

        conn = _get_conn()
        try:
            # 获取最近1000条的 simhash 值比较
            rows = conn.execute(
                "SELECT simhash FROM subscription_history "
                "WHERE simhash IS NOT NULL "
                "ORDER BY id DESC LIMIT 1000"
            ).fetchall()

            for row in rows:
                if SimHash.is_near_duplicate(new_hash, row[0]):
                    return True
            return False
        finally:
            conn.close()
```

### 10.1 AI 智能爬虫（LLM-Powered Crawler）

传统爬虫依赖手写 CSS 选择器/XPath，网站改版就失效。AI 爬虫用大语言模型理解页面结构，**自动适应网站变化**。

| 工具 | GitHub Stars | 核心能力 | 适合场景 |
|------|:-----------:|----------|----------|
| [Firecrawl](https://github.com/firecrawl/firecrawl) | 81K+ | API服务，将任意URL转为LLM-ready Markdown/JSON，自带导航Agent | 快速集成，不想维护爬虫逻辑 |
| [Crawl4AI](https://github.com/unclecode/crawl4ai) | 60K+ | 本地部署，自适应DOM变化，支持Ollama本地模型 | 数据不出本地、深度定制 |
| [ScrapeGraphAI](https://github.com/VinciGit00/Scrapegraph-ai) | - | 自然语言描述想要的数据→自动提取 | 非技术人员也能配置采集 |
| [Browser Use](https://github.com/browser-use/browser-use) | 50K+ | 自主浏览器控制，模拟真人操作 | 复杂交互场景 |

**我们项目可以怎么用**：
```python
# 示例：用 Crawl4AI 替代手写爬虫
from crawl4ai import WebCrawler

crawler = WebCrawler()
result = crawler.run(url="https://36kr.com/p/123456")
print(result.markdown)   # 干净的正文 Markdown
print(result.metadata)   # 标题、作者、时间等
```

**价值**：不用再为每个新闻源写不同的解析规则，AI 自动理解页面结构。

> 参考：[Crawl4AI vs Firecrawl 对比](https://www.capsolver.com/blog/AI/crawl4ai-vs-firecrawl) | [AI Web Scraping 2026](https://www.morphllm.com/ai-web-scraping)

### 10.2 视频平台采集（YouTube / Bilibili / 抖音）

```python
import yt_dlp

def get_video_metadata(url):
    """提取视频元数据（不下载视频文件）"""
    opts = {'quiet': True, 'skip_download': True, 'no_warnings': True}
    with yt_dlp.YoutubeDL(opts) as ydl:
        info = ydl.extract_info(url, download=False)
        return {
            'title': info.get('title'),
            'description': info.get('description'),
            'duration': info.get('duration'),        # 时长（秒）
            'view_count': info.get('view_count'),
            'like_count': info.get('like_count'),
            'upload_date': info.get('upload_date'),
            'uploader': info.get('uploader'),
            'thumbnail': info.get('thumbnail'),
            'tags': info.get('tags', []),
            'categories': info.get('categories', []),
        }

# 支持的平台（yt-dlp 支持 1000+ 站点）
get_video_metadata("https://www.youtube.com/watch?v=xxx")
get_video_metadata("https://www.bilibili.com/video/BVxxx")
```

### 10.3 搜索引擎结果采集（Google / Bing / 百度）

通过搜索引擎 API 按关键词主动发现新内容，而非被动等待 RSS 更新。

| 方案 | 价格 | 优势 |
|------|------|------|
| [SerpAPI](https://serpapi.com/) | $50/月 5000次 | 最成熟，支持80+搜索引擎，结构化JSON |
| [Serper.dev](https://serper.dev/) | $50/月 50000次 | 性价比最高，速度最快 |
| [DataForSEO](https://dataforseo.com/) | $0.0006/次 | 大规模最便宜 |
| 自建（requests + BeautifulSoup） | 免费 | 易被封IP，需代理池 |

**我们项目可以怎么用**：
```python
# 示例：用 Serper.dev 搜索最新AI新闻
import requests

def search_news(keyword, num=10):
    resp = requests.post("https://google.serper.dev/news", json={
        "q": keyword,
        "gl": "cn",
        "hl": "zh-cn",
        "num": num
    }, headers={"X-API-KEY": "your_key"})
    return resp.json().get("news", [])

# 每小时搜索热门关键词，发现新文章
for article in search_news("人工智能 最新进展"):
    print(f"{article['title']} - {article['link']}")
```

**价值**：从"等待RSS推送" 升级为 "主动搜索发现"，覆盖没有RSS的站点。

> 参考：[SerpAPI](https://serpapi.com/) | [Best SERP APIs 2026](https://www.scrapingdog.com/blog/best-serp-apis/)

### 10.4 JS渲染页面采集（Playwright / Puppeteer）

94% 的现代网站依赖 JavaScript 渲染，传统 requests 库无法获取内容。

```python
# 使用 Playwright（Python版）采集 JS 渲染页面
from playwright.sync_api import sync_playwright

def scrape_js_page(url):
    with sync_playwright() as p:
        browser = p.chromium.launch(headless=True)
        page = browser.new_page()
        page.goto(url, wait_until='networkidle')

        # 等待内容加载完成
        page.wait_for_selector('.article-content')

        title = page.title()
        content = page.inner_text('.article-content')

        browser.close()
        return {'title': title, 'content': content}
```

| 工具 | 语言 | 浏览器支持 | 适合场景 |
|------|------|:----------:|----------|
| [Playwright](https://playwright.dev/python/) | Python/Node/Java | Chromium+Firefox+WebKit | 跨浏览器、企业级 |
| [Puppeteer](https://pptr.dev/) | Node.js | Chrome | Chrome专用、速度快 |
| [Selenium](https://www.selenium.dev/) | 多语言 | 全部 | 老牌、生态丰富 |

**我们项目可以怎么用**：
- 对无 RSS 的 JS 渲染站点（如某些论坛、SPA 应用），用 Playwright 采集
- 替代现有 fetch_news.py 中对 JS 站点采集失败的问题

> 参考：[Playwright Scraping Guide](https://www.browserless.io/blog/scraping-with-playwright-a-developer-s-guide-to-scalable-undetectable-data-extraction)
