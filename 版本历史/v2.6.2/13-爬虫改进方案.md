## 1. 现状问题汇总

### 1.1 致命问题

| 编号 | 问题 | 影响 | 现有代码位置 |
|:----:|------|------|-------------|
| W1 | subscription_history 无去重 | 每次检测到变化，RSS 全部条目重复插入，数据膨胀 | `subscribe_manager.py` → `save_update_history()` |
| W2 | 订阅源重复4份 | subscribe.db 有20条记录，实际只有5个源，每次采集重复4倍工作量 | `data/subscribe.db` → `subscription` 表 |

### 1.2 高优先级问题

| 编号 | 问题 | 影响 |
|:----:|------|------|
| W3 | 指纹机制不可靠 | 前5条标题拼接做指纹，feed 排序变化就误判"有更新"，导致全量重插 |
| W5 | 历史数据 link 全为 NULL | 旧数据24条 link 字段为空，无法用于去重 |
| W8 | 无自动调度 | 没有定时采集机制，完全靠管理员手动触发 |
| W9 | fetch_news.py 两份不一致 | `crawler_service.py` 导入的是根目录旧版（30条上限），模块目录新版（200条+均衡选取）从未被 API 调用 |

### 1.3 中低优先级问题

| 编号 | 问题 | 影响 |
|:----:|------|------|
| W4 | 新闻标题去重截断30字 | 相似标题可能误判重复 |

| W10 | 错误处理缺陷 | DB连接可能泄漏、异常静默吞掉、无重试 |
| W11 | 无连接池 | 每次操作独立开关 SQLite 连接 |

## 2. 改进方案

### 阶段一：订阅爬虫修复（核心，建议立即做）

#### 2.1 W1修复：subscription_history 去重

**方案**：插入前按 `link` 查重，已存在则跳过。

**修改文件**：`subscribe_manager.py` → `save_update_history()`

```python
def save_update_history(sub_id, name, items):
    conn = init_db()
    c = conn.cursor()
    # ... CREATE TABLE ...

    now = datetime.now().strftime("%Y-%m-%d %H:%M:%S")
    inserted = 0

    for item in items:
        link = item.get('link', '')
        title = item.get('title', '')

        # 按 link 去重（link非空时），否则按 title+sub_id 去重
        if link:
            c.execute("SELECT id FROM subscription_history WHERE link = ? AND sub_id = ?",
                      (link, sub_id))
        else:
            c.execute("SELECT id FROM subscription_history WHERE title = ? AND sub_id = ?",
                      (title, sub_id))

        if c.fetchone():
            continue  # 已存在，跳过

        c.execute('''INSERT INTO subscription_history ...''', (...))
        inserted += 1

    conn.commit()
    conn.close()
    print(f"  💾 新增 {inserted} 条，跳过 {len(items) - inserted} 条重复")
```

**预期效果**：
- 同一篇文章不会重复入库
- 采集日志清晰显示新增/跳过数量

#### 2.2 W2修复：清理重复订阅源

**方案**：一次性数据清理脚本 + 添加唯一约束。

**操作步骤**：

```sql
-- 1. 查看重复
SELECT url, COUNT(*) as cnt FROM subscription GROUP BY url HAVING cnt > 1;

-- 2. 保留每个URL最早的一条，删除其余
DELETE FROM subscription WHERE id NOT IN (
    SELECT MIN(id) FROM subscription GROUP BY url
);

-- 3. 添加唯一索引（防止未来重复）
CREATE UNIQUE INDEX IF NOT EXISTS idx_subscription_url ON subscription(url);
```

**同时修改 DAL**：`subscribe_dal.py` → `create_subscription()` 添加 URL 重复检查。

#### 2.3 W3修复：改进更新检测机制

**现有方案**：前5条标题拼接做指纹 → 不可靠
**改进方案**：逐条对比 link，只保存新条目。

```python
def check_for_updates(sub_id):
    # ... 获取订阅信息 ...

    if sub_type == 'rss':
        items = parse_rss_feed(url)
        if not items:
            return False

        # 获取已存在的 link 集合
        c.execute("SELECT link FROM subscription_history WHERE sub_id = ?", (sub_id,))
        existing_links = {row[0] for row in c.fetchall() if row[0]}

        # 筛选出真正的新条目
        new_items = [item for item in items if item.get('link') and item['link'] not in existing_links]

        if not new_items:
            print(f"  ⏹️  无新内容")
            # 更新检查时间
            c.execute("UPDATE subscription SET last_check = ? WHERE id = ?", (now, sub_id))
            conn.commit()
            conn.close()
            return False

        print(f"  ✅ 发现 {len(new_items)} 条新内容（共 {len(items)} 条）")
        c.execute("UPDATE subscription SET last_check = ? WHERE id = ?", (now, sub_id))
        conn.commit()
        save_update_history(sub_id, name, new_items)  # 只保存新条目
        conn.close()
        return True
```

**优势**：
- 不再依赖脆弱的指纹机制
- 只插入真正的新文章
- 不受 feed 排序变化影响

### 阶段三：自动调度（可选，建议做）

**方案A：Flask 内置 APScheduler**（推荐）

```python
from apscheduler.schedulers.background import BackgroundScheduler

scheduler = BackgroundScheduler()

# 新闻采集：每2小时

# RSS订阅采集：每30分钟

scheduler.start()
```

**方案B：简单 Timer 循环**（零依赖）

```python
# 在 crawler_service.py 中添加
import threading

def _schedule_loop():
    """简单的定时采集循环"""
    while True:
        try:
            check_all_subscriptions()  # RSS采集
        except Exception as e:
            print(f"定时采集失败: {e}")
        threading.Event().wait(1800)  # 每30分钟

# 启动时调用
def start_scheduler():
    t = threading.Thread(target=_schedule_loop, daemon=True)
    t.start()
```

```python
def parse_rss_feed(url, timeout=15, retries=2):
    for attempt in range(retries + 1):
        try:
            feed = feedparser.parse(url, request_headers={
                'User-Agent': 'Mozilla/5.0 ...'
            })
            if feed.bozo and not feed.entries:
                raise Exception(f"Feed parse error: {feed.bozo_exception}")
            # ... 解析逻辑 ...
            return items
        except Exception as e:
            if attempt < retries:
                print(f"  ⚠️ 第{attempt+1}次失败，重试中... ({e})")
                time.sleep(2)
            else:
                print(f"  ❌ RSS解析最终失败：{url} - {e}")
                return None
```
