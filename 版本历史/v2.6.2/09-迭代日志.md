##### API 接口（前端对接用）

**1. 获取订阅内容列表**

`GET /api/subscribe/feed`

| 参数 | 类型 | 必填 | 说明 |
|------|------|:----:|------|
| page | int | 否 | 页码，默认 1 |
| per_page | int | 否 | 每页数量，默认 20 |
| source | string | 否 | 按订阅源名称筛选 |

响应：
```json
{
  "code": 200,
  "data": [
    {
      "id": 1,
      "sub_id": 3,
      "sub_name": "36氪",
      "title": "AI大模型最新进展",
      "summary": "文章摘要内容，最多500字...",
      "content": "完整内容",
      "link": "https://36kr.com/p/123456",
      "detected_at": "2026-03-13 15:37:38",
      "source_url": "https://36kr.com/feed",
      "sub_type": "rss"
    }
  ],
  "total": 24,
  "page": 1,
  "per_page": 20
}
```

**2. 获取单条订阅详情**

`GET /api/subscribe/{id}`

响应：
```json
{
  "code": 200,
  "data": {
    "id": 1,
    "sub_id": 3,
    "sub_name": "36氪",
    "title": "AI大模型最新进展",
    "summary": "文章摘要...",
    "content": "完整内容...",
    "link": "https://36kr.com/p/123456",
    "detected_at": "2026-03-13 15:37:38",
    "source_url": "https://36kr.com/feed",
    "sub_type": "rss"
  }
}
```

404时返回：`{ "code": 404, "message": "内容不存在" }`

**3. 获取订阅源列表（筛选用）**

`GET /api/subscribe/sources`

响应：
```json
{
  "code": 200,
  "data": ["36氪", "知乎热榜", "少数派"]
}
```

##### 新旧字段对比

| 字段 | 旧版 | 新版（v2.6.2） |
|------|------|----------------|
| title | 无，需从content纯文本提取 | 直接返回文章标题 |
| summary | 无 | 直接返回文章摘要（最多500字） |
| link | 无，只有source_url（RSS源地址） | 直接返回文章原文链接 |
| content | RSS纯文本dump | 结构化内容 |

##### 前端取值逻辑

```javascript
// 标题
const title = item.title || '无标题';
// 摘要
const summary = item.summary || '';
// 查看原文链接（直接跳转文章页，不再跳到RSS源）
const articleLink = item.link || item.source_url;
```

##### 注意事项
- 需要重新执行一次采集，旧数据的 title/summary/link 字段为空
- 前端已有的回退逻辑（从content提取）可以兼容旧数据
- DAL/Service/API路由层此前已搭好骨架，本次无需修改

**问题12：采集日志统计数据为0，详情为空**
- **发现者**：用户反馈
- **问题描述**：执行采集任务后，采集日志列表显示正常，但"总数"、"新增"、"更新"列全为0，且"详情"列显示"-"（无数据）
- **根因分析**：
  1. **fetch_news.py:770** 原有判断 `if row and row[1]: continue` 太严格——只要新闻已存在且有 summary 就直接跳过整个循环
  2. 导致 `new_count`、`updated_count` 始终为 0，`news_detail_list` 始终为空数组
- **修复方案**：
  - 移除 `fetch_news.py:770` 的 `if row and row[1]: continue` 判断
  - 调整代码顺序，把 `print` 语句、RSS时间/summary 处理、详情获取等逻辑移到判断之前
  - 确保无论新闻是否已存在，都会执行到计数和 detail 记录逻辑
- **修改文件**：`g:/news-system/modules/news_crawler/crawlers/fetch_news.py`
- **代码变更**：
  ```python
  # 修改前
  for n in selected:
      c.execute("SELECT id, summary FROM news WHERE link = ?", (n["link"],))
      row = c.fetchone()
      if row and row[1]:  # 已有完整数据
          continue  # ← 直接跳过，导致计数和detail都是空
      print(f"-> {n['title'][:40]}")
      ...

  # 修改后
  for n in selected:
      c.execute("SELECT id, summary FROM news WHERE link = ?", (n["link"],))
      row = c.fetchone()
      print(f"-> {n['title'][:40]}")
      ...  # 先执行所有处理逻辑
      if row:  # 更新
          updated_count += 1
      else:  # 新增
          new_count += 1
      news_detail_list.append(...)  # 无论新旧都记录detail
  ```

**问题13：RSS采集失败，gbk编码错误**
- **发现者**：用户反馈
- **问题描述**：执行全量采集后，RSS采集显示 "failed: 'gbk' codec can't encode character '\\u2705' in position 2: illegal multibyte sequence"
- **根因分析**：
  - `subscribe_manager.py` 中使用了 emoji 字符（✅ ⚠️ ⏹️ 💾）
  - Windows 默认编码为 GBK，无法输出 Unicode emoji，导致编码错误
- **修复方案**：
  - 将所有 emoji 替换为文字标签
  - 同时修复根目录和 scripts 目录下的两个版本
- **修改文件**：
  - `g:/news-system/subscribe_manager.py`
  - `g:/news-system/scripts/subscribe_manager.py`
- **替换对照表**：

  | ⚠️ | [WARN] | 警告 |

## 技术建议

### 推荐库
- **RSS解析**: feedparser (Python)
- **HTML清理**: beautifulsoup4 + lxml
- **缩略图提取**: 从enclosure或img标签中提取

### 参考实现
```python
import feedparser
from bs4 import BeautifulSoup

def parse_rss_content(xml_content):
    feed = feedparser.parse(xml_content)
    for entry in feed.entries:
        title = entry.title
        summary = entry.get('summary', '')
        content = entry.get('content', [{}])[0].get('value', summary)
        link = entry.link
        pub_date = entry.get('published', '')
        author = entry.get('author', '')
        
        # 提取缩略图
        thumbnail = None
        if 'media_thumbnail' in entry:
            thumbnail = entry.media_thumbnail[0]['url']
        elif 'enclosures' in entry and entry.enclosures:
            thumbnail = entry.enclosures[0].href
        
        yield {
            'title': title,
            'summary': summary,
            'content': content,
            'link': link,
            'pub_date': pub_date,
            'author': author,
            'thumbnail': thumbnail
        }
```
