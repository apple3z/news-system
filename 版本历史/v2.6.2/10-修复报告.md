# v2.6.2 问题修复报告

## 问题概述

用户反馈了两个严重问题：
1. 前台的 Skill 工具排行榜不见了
2. 订阅管理没有爬回来的数据

## 问题根因分析

### 问题 1：Skills 排行榜为空

**根因**：Skills 数据采集脚本（fetch_skills.py）在插入数据时没有设置 `stars` 和 `downloads` 字段的值，导致这些字段为 NULL。

排行榜查询 SQL 条件：
```sql
SELECT id, name, description, category, stars, downloads 
FROM skills WHERE stars IS NOT NULL ORDER BY stars DESC LIMIT ?
```

由于所有记录的 stars 都是 NULL，查询返回空结果。

**修复方案**：
1. 修改 `fetch_skills.py`，在插入数据时设置 stars 和 downloads 默认值为 0
2. 清空现有数据并重新采集
3. 更新现有记录的 stars 和 downloads 字段

**修复代码**：
```python
# 在 INSERT 语句中添加 stars 和 downloads 字段
conn.execute("""INSERT INTO skills (name,owner,title,description,source,url,download_url,github_url,category,features,capabilities,skill_level,chinese_intro,readme_content,stars,downloads,created_at) 
    VALUES (?,?,?,?,?,?,?,?,?,?,?,?,?,?,?,?,?)""",
    (n, o, n, n[:50], 'clawhub', f"https://clawhub.ai/skill/{o}/{n}",
     f"https://github.com/{o}/{n}/zipball", f"https://github.com/{o}/{n}",
     category, ft, ft, lv, intro, rd[:3000], 0, 0, dt.now().strftime("%Y-%m-%d")))
```

### 问题 2：订阅内容为空

**根因**：有多个 Flask 进程同时运行在 5000 端口，导致 HTTP 请求被路由到错误的进程。

数据库查询正常：
- subscription_history 表有 24 条记录
- DAL 层和服务层函数都正常返回数据
- Flask 测试客户端测试 API 返回正确数据

但通过 HTTP 请求 API 时返回空数据，因为请求被路由到了旧的 Flask 进程。

**修复方案**：
1. 停止所有 Python 进程
2. 重新启动单个 Flask 应用

**操作命令**：
```bash
taskkill /F /IM python.exe
python app.py
```

### 问题 3：采集工作台添加数据源按钮点击无弹窗

**根因**：NewsAdmin.js中modal-dialog组件使用v-if控制显示，但Modal组件需要通过:show prop控制。Modal组件内部通过`:show` prop来添加/移除`show` CSS类来控制显示。

**修复方案**：
- 修改 `NewsAdmin.js`，将 `v-if="showModal"` 改为 `:show="showModal"`

**修复代码**：
```javascript
// 修改前
<modal-dialog v-if="showModal" @close="showModal = false"
              :title="editMode ? '编辑数据源' : '添加数据源'">

// 修改后
<modal-dialog :show="showModal" @close="showModal = false"
              :title="editMode ? '编辑数据源' : '添加数据源'">
```

### 问题 4：订阅详情页白屏

**根因**：SubscribeDetailView.js存在语法错误（缺失闭括号），导致整个文件无法正确解析。

**修复方案**：
1. 重写 SubscribeDetailView.js
2. 修复语法错误
3. 优化 getTitle/getSummary/extractArticleLink 方法，优先使用 API 返回的结构化字段

**修复代码**：
```javascript
// 优化前
getTitle() {
    // 从content提取标题
}

// 优化后
getTitle() {
    if (this.item.title) return this.item.title;
    // 回退逻辑：从content提取
}
```

### 问题 5：后台RSS源列表为空

**根因**：数据源管理使用 data_sources 表，而订阅源存放在 subscription 表中，两者是分开的。data_sources 表初始只有新闻源和Skill源，没有RSS源。

**修复方案**：
1. 手动向 data_sources 表添加5个RSS源
2. 验证RSS源列表正常显示

**添加的RSS源**：
- 科技美学
- 36氪
- 虎嗅网
- 机器之心
- 量子位
