## 四、部署架构

### 4.1 目录结构

```
news_system/
├── app.py                      # Flask Web应用入口 (v2.2)
├── fetch_news.py               # 新闻采集脚本
├── fetch_skills.py            # Skills采集脚本
├── cover_manager.py           # 封面图管理
├── subscribe_manager.py       # 订阅管理
│
├── data/                      # 数据目录
│   ├── news.db               # 新闻数据库 (39条)
│   ├── skills.db             # Skills数据库 (10条)
│   ├── subscribe.db          # 订阅数据库
│   └── covers/              # 默认封面图库
│
├── docs/                      # 文档目录 (Wiki源)
│   ├── PRD.md                # 产品需求文档
│   ├── ARCH.md               # 架构设计文档
│   ├── TDD.md                # 技术设计文档
│   └── DEVELOPMENT.md       # 开发文档
│
├── logs/                      # 日志
│   └── fetch.log             # 采集日志
│
└── scripts/                   # 辅助脚本
    └── cron_fetch.sh         # 定时采集脚本
```

## 五、接口设计

### 5.1 Web路由

| 方法 | 路径 | 说明 | 状态 |
|------|------|------|------|
| GET | / | 新闻首页 | ✅ |
| GET | /news/\<id\> | 新闻详情 | ✅ |
| GET | /skills | Skills列表 | ✅ |
| GET | /skill/\<id\> | Skill详情 | ✅ |
| GET | /subscribe | 订阅管理 | ✅ |
| GET | /dev | 研发管理(版本+Wiki) | 🆕 |
| GET | /wiki/\<name\> | Wiki文档预览 | 🆕 |
| POST | /wiki/\<name\> | Wiki文档保存 | 🆕 |
| POST | /api/subscribe | 添加订阅 | ✅ |
| DELETE | /api/subscribe/\<id\> | 删除订阅 | ✅ |
| GET | /api/refresh | 刷新新闻 | ✅ |

### 5.2 Wiki接口

**获取文档列表**:
```json
GET /api/wiki/list
{
  "code": 200,
  "data": [
    {"name": "PRD.md", "title": "产品需求文档"},
    {"name": "ARCH.md", "title": "架构设计文档"},
    {"name": "TDD.md", "title": "技术设计文档"},
    {"name": "DEVELOPMENT.md", "title": "开发文档"}
  ]
}
```

**获取文档内容**:
```json
GET /api/wiki/PRD.md
{
  "code": 200,
  "data": {
    "name": "PRD.md",
    "content": "# 📋 PRD产品需求文档..."
  }
}
```

**保存文档**:
```json
POST /api/wiki/PRD.md
{
  "content": "# 📋 PRD产品需求文档...\n(新内容)"
}

{
  "code": 200,
  "msg": "保存成功"
}
```
