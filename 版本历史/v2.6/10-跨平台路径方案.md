## 一、问题背景

### 1.1 问题描述

在 Windows 环境下运行时，发现以下问题：
- 新闻功能无法显示数据
- Skills 功能无法正常运行
- 订阅管理功能失效

### 1.2 根本原因

**Linux 路径硬编码** - 所有脚本都使用了 Linux 绝对路径：
```python
# ❌ 错误示例
DB_FILE = "/home/zhang/.copaw/news_system/data/news.db"
BASE_DIR = "/home/zhang/news_system"
```

在 Windows 环境下无法访问这些路径，导致：
- 数据库连接失败
- 数据无法读取
- 功能完全失效

### 1.3 研发管理规范的要求

根据 [`研发管理规范.md`](file://g:\news-system\news-system\研发管理规范.md)：

> **第 2.4 节 - 项目路径**
> - News System: `/home/zhang/news_system/` (Linux 部署路径)
> 
> **第 3.1 节 - 目录结构**
> - 版本管理：`版本管理/`
> - 文档中心：`文档中心/`

**问题**：规范中只记录了 Linux 路径，但代码需要同时支持 Windows 和 Linux。

## 二、解决方案

### 2.1 统一跨平台路径工具

创建 [`path_utils.py`](file://g:\news-system\news-system\path_utils.py) 模块，提供统一的路径管理：

```python
#!/usr/bin/env python3
"""跨平台路径工具模块"""

import os
import platform


def get_project_root():
    """智能获取项目根目录，自动适配 Windows/Linux"""
    current_dir = os.path.dirname(os.path.abspath(__file__))
    
    # 检查是否有版本历史/文档中心
    if os.path.exists(os.path.join(current_dir, '版本历史')) or \
       os.path.exists(os.path.join(current_dir, '文档中心')):
        return current_dir
    
    # Linux 部署路径
    if platform.system() == 'Linux':
        linux_paths = [
            '/home/zhang/news_system',
            '/opt/news_system',
            '/var/www/news_system'
        ]
        for path in linux_paths:
            if os.path.exists(path):
                return path
    
    return current_dir


def get_data_dir():
    """获取数据目录路径"""
    return os.path.join(get_project_root(), 'data')


def get_db_path(db_name):
    """获取数据库文件路径"""
    return os.path.join(get_data_dir(), db_name)


def ensure_data_dir():
    """确保数据目录存在"""
    data_dir = get_data_dir()
    if not os.path.exists(data_dir):
        os.makedirs(data_dir, exist_ok=True)
    return data_dir


# 预定义常用路径
PROJECT_ROOT = get_project_root()
DATA_DIR = get_data_dir()
NEWS_DB = get_db_path('news.db')
SKILLS_DB = get_db_path('skills.db')
SUBSCRIBE_DB = get_db_path('subscribe.db')
```

## 三、技术要点

### 3.1 路径策略

1. **优先使用脚本所在目录** - 最准确
2. **检测特征目录** - 版本历史/文档中心
3. **检查 Linux 部署路径** - 多个候选路径
4. **降级到当前目录** - 最后方案

### 3.2 跨平台兼容

- ✅ 使用 `os.path.join()` - 自动处理路径分隔符
- ✅ 使用 `os.path.dirname()` - 获取目录名
- ✅ 使用 `os.path.abspath()` - 获取绝对路径
- ✅ 使用 `platform.system()` - 检测操作系统
- ✅ 使用 `os.makedirs(exist_ok=True)` - 创建目录

### 3.3 数据库表创建

```python
# ❌ 错误：每次都尝试创建表
conn.execute("""CREATE TABLE skills (...)""")

# ✅ 正确：只在不存在时创建
conn.execute("""CREATE TABLE IF NOT EXISTS skills (...)""")
```
