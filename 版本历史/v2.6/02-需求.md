## 一、产品定位

### 1.1 产品愿景
打造一款**功能完善、体验优秀、稳定可靠**的 AI 科技新闻与 Skills 工具门户，为用户提供：
- 📰 **高质量的 AI 科技新闻聚合**
- 🛠️ **实用的 AI Skills 工具库**
- 📥 **便捷的订阅管理功能**

### 1.2 目标用户
| 用户类型 | 需求场景 | 核心价值 |
|---------|---------|---------|
| **AI 从业者** | 获取最新 AI 科技动态 | 节省时间，不错过重要信息 |
| **开发者** | 发现实用 AI 工具 | 快速找到合适的 Skills |
| **技术管理者** | 跟踪行业趋势 | 把握技术发展方向 |

### 1.3 v2.6 版本目标

**"修复 BUG，功能完善，体验优化"**

#### 核心目标（按优先级）
1. 🔴 **修复三大模块的严重 BUG** - 新闻、Skills、订阅必须能正常运行
2. ✅ **首页（新闻）功能完善** - 增加新闻源、分类、搜索
3. ✅ **Skills 工具功能完善** - 分类体系、搜索、数据采集扩展
4. ✅ **订阅管理功能完善** - 自动检测、预览、类型扩展

```python
# 后台定时任务
def check_subscription_updates():
    for sub in active_subscriptions:
        new_content = fetch_url(sub.url)
        if new_content != sub.last_content:
            save_update(sub.id, new_content)
            notify_user(sub.id)
```

## 六、接口设计

### 6.1 新闻模块 API

```python
# 获取新闻列表（支持分类、搜索、分页）
GET /api/news?category=AI 大模型&page=1&size=20&keyword=搜索词

# 获取新闻详情
GET /api/news/{id}

# 收藏/取消收藏
POST /api/news/{id}/favorite
DELETE /api/news/{id}/favorite

# 获取收藏列表
GET /api/user/favorites

# 管理新闻源
GET /api/news/sources
POST /api/news/sources
PUT /api/news/sources/{id}
DELETE /api/news/sources/{id}

# 手动触发采集
POST /api/news/fetch
```

### 6.2 Skills 模块 API

```python
# 获取 Skills 列表（支持分类、搜索）
GET /api/skills?category=AI 工具&page=1&keyword=搜索词

GET /api/skills/{id}

# 评分
POST /api/skills/{id}/rating
{

    "comment": "很好用"
}

# 获取评论
GET /api/skills/{id}/reviews

# 提交教程
POST /api/skills/{id}/tutorial

# 触发 Skills 采集
POST /api/skills/fetch
```
