## 二、问题详情

### BUG-NEWS-P0-001: 新闻功能完全无法正常运行

**发现日期**: 2026-03-09  
**报告人**: 张振中  
**优先级**: 🔴 P0（阻塞性）

#### 问题描述
- 新闻首页无法显示新闻列表
- 新闻详情页无法打开
- 新闻采集脚本执行失败

#### 根本原因
1. `fetch_news.py` 使用 Linux 路径：`/home/zhang/.copaw/news_system/data/news.db`
2. `data/` 目录不存在
3. 数据库表未初始化

#### 修复方案
1. 修改路径为跨平台路径：
   ```python
   BASE_DIR = os.path.dirname(os.path.abspath(__file__))
   DB_FILE = os.path.join(BASE_DIR, 'data', 'news.db')
   ```
2. 创建 `data/` 目录
3. 运行 `python init_db.py` 初始化数据库

#### 修复结果
- ✅ 首页正常显示（Status 200）
- ✅ 详情页正常打开
- ✅ 采集脚本可正常执行
- ✅ 无 Python 报错
- ✅ 无 JavaScript 报错

**验收人**: 瑞贝卡  
**验收日期**: 2026-03-09

## 五、经验教训

### 技术层面
1. **必须使用跨平台路径**
   - 避免硬编码绝对路径
   - 使用 `os.path.join()` 和 `os.path.dirname()`
   - 使用相对路径或环境变量

2. **数据库初始化很重要**
   - 需要自动检测数据库是否存在
   - 提供初始化脚本
   - 在应用启动时自动检查

3. **测试要全面**
   - 每个模块单独测试
   - 跨平台测试（Windows/Linux）
   - 功能测试 + 集成测试
