## 二、问题详情

### BUG-001: 文档列表API返回空数据

**严重程度**: 🔴 高

**问题描述**:
- 访问 `/api/v2/doc/list?path=文档中心` 返回空数组
- 访问 `/api/v2/doc/list?path=` 也返回空数组
- 实际 `文档中心/` 目录下有 5 个文件

**复现步骤**:
```bash
curl "http://localhost:5000/api/v2/doc/list?path=文档中心"
# 返回: {"code":200,"data":[]}
```

**根本原因**:
- `version_manager.py` 的 `_resolve_path()` 函数只处理以 `版本历史/` 或 `文档中心/` 开头的路径
- 空路径 `path=""` 没有被正确处理，返回 `None`
- 导致 `list_documents()` 返回空数组

**影响范围**:
- 版本历史页面无法显示目录结构
- 文档中心页面无法显示目录结构

**修复建议**:
在 `version_manager.py` 的 `_resolve_path()` 函数中添加空路径处理：
```python
def _resolve_path(path):
    if not path:
        return DOCS_DIR  # 默认返回文档中心目录
    # ... 其他逻辑
```

### BUG-002: 订阅管理API路径不一致

**严重程度**: 🟡 中

**问题描述**:
- 前端代码可能调用 `/api/subscribe/list` 获取订阅列表
- 后端实际提供的API是 `/api/subscribe` (POST)

**根本原因**:
- subscribe_api.py 提供的端点：
  - `POST /api/subscribe` - 添加订阅
  - `DELETE /api/subscribe/<id>` - 删除订阅
  - `POST /api/subscribe/check` - 检查订阅
- 缺少获取订阅列表的API

**修复建议**:
在 `subscribe_api.py` 中添加获取列表的API：
```python
@news_crawler_bp.route('/api/subscribe/list', methods=['GET'])
def api_subscribe_list():
    # 获取订阅列表逻辑
```

### BUG-003: 系统管理页面标签页点击无反应

**严重程度**: 🔴 高

**问题描述**:
- 系统管理页面中，"版本历史"和"文档中心"标签点击无反应
- "新闻管理"、"Skill管理"、"订阅管理"标签也无法切换

**当前状态**: 已修复，需用户验证

### BUG-006: 前端CDN依赖问题

**严重程度**: 🟢 低

**问题描述**:
- 前端依赖 Vue 3 和 Vue Router 的 CDN
- 如果用户无法访问 unpkg.com，页面将无法正常工作

**修复建议**:
- 考虑将Vue和Vue Router下载到本地
- 或使用其他更稳定的CDN
