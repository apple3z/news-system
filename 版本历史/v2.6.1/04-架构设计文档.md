### 2.1 架构设计

前端采用 **Vue 3 SPA (Single Page Application)** 架构，所有页面通过 Vue Router 进行无刷新切换。


## 4. 分层架构设计

### 4.1 三层架构

每个模块内部采用严格的分层架构：

```
Routes (API)  ──>  Services (业务逻辑)  ──>  DAL (数据访问)  ──>  SQLite/文件系统
   │                    │                       │
  Flask              纯函数                 sqlite3
  request/           无HTTP依赖            直接SQL
  response
```

#### 4.1.1 Routes 层 (API 路由)

职责：
- 解析 HTTP 请求参数
- 调用 Service 层函数
- 封装 JSON 响应

示例 (`news_api.py`)：
```python
@news_crawler_bp.route('/api/news/search')
def api_news_search():
    result = news_service.search_news(
        keyword=request.args.get('keyword', ''),
        category=request.args.get('category', ''),
        ...
    )
    return jsonify({'code': 200, 'data': result['data'], 'total': result['total']})
```

#### 4.1.2 Services 层 (业务逻辑)

职责：
- 封装业务逻辑
- 不依赖 Flask 对象
- 可被其他模块调用

示例 (`news_service.py`)：
```python
def search_news(keyword='', category='', source='', ...):
    """搜索新闻 - 纯业务逻辑，无 HTTP 依赖"""
    return news_dal.search_news(keyword=keyword, category=category, ...)
```

#### 4.1.3 DAL 层 (数据访问)

职责：
- 直接操作数据库
- 封装 SQL 语句
- 返回 dict/list 结构

示例 (`news_dal.py`)：
```python
def search_news(keyword='', category='', ...):
    conn = sqlite3.connect(NEWS_DB)
    c = conn.cursor()
    c.execute("SELECT * FROM news WHERE ...", params)
    rows = c.fetchall()
    conn.close()
    return [{'id': row[0], 'title': row[1], ...} for row in rows]
```

## 5. 代理层设计 (Proxy)

### 5.1 设计目的

代理层 (`proxy/registry.py`) 实现**模块间解耦通信**，让模块之间无需直接 import 即可调用服务。

### 5.2 核心 API

```python
# 注册服务
def register(name, func):
    """注册服务函数"""
    _registry[name] = func

# 调用服务
def call(name, **kwargs):
    """通过名称调用已注册的服务"""
    if name not in _registry:
        raise KeyError(f"Service '{name}' not registered")
    return _registry[name](**kwargs)

# 获取服务
def get(name):
    """获取已注册的函数引用"""
    return _registry.get(name)
```

### 5.3 使用示例

**模块A (news_crawler) 注册服务：**

```python
# modules/news_crawler/__init__.py
from proxy.registry import register
from modules.news_crawler.services.news_service import search_by_keywords

register('news.search_by_keywords', search_by_keywords)
```

**模块B (subscribe) 调用服务：**

```python
# modules/news_crawler/services/subscribe_service.py
from proxy import call

# 无需 import news_crawler 模块
results = call('news.search_by_keywords', keywords=['AI', 'ML'])
```

### 5.4 已注册服务列表

| 服务名 | 提供者 | 用途 |
|--------|--------|------|
| `news.search_by_keywords` | news_crawler | subscribe 模块检查关键词匹配新闻 |

## 6. 版本管理设计

### 6.1 版本管理架构

版本管理由 `version_manager.py` 实现：

```
┌─────────────────────────────────────────┐
│           version_manager.py             │
│  (版本号递增 + 历史记录 + 文件操作)      │
└─────────────────────────────────────────┘
                │
                ▼
┌─────────────────────────────────────────┐
│         modules/sys_admin/dal/           │
│            version_dal.py                │
│     (DAL 封装，调用 version_manager)     │
└─────────────────────────────────────────┘
                │
                ▼
┌─────────────────────────────────────────┐
│        modules/sys_admin/services/       │
│             doc_service.py               │
│          (业务逻辑封装)                   │
└─────────────────────────────────────────┘
                │
                ▼
┌─────────────────────────────────────────┐
│         modules/sys_admin/routes/        │
│              doc_api.py                   │
│            (API 端点)                     │
└─────────────────────────────────────────┘
```

### 6.2 版本号规则

采用语义化版本号格式：`主版本.次版本.修订号` (Major.Minor.Patch)

- **Major (主版本)**: 重大架构变更
- **Minor (次版本)**: 功能新增
- **Patch (修订号)**: 每次保存自动 +1

### 6.3 版本元数据存储

版本元数据存储在 `doc/.versions.json`：

```json
{
  "研发管理规范.md": {
    "major": 1,
    "minor": 0,
    "patch": 5,
    "version": "1.0.5",
    "history": [
      {"version": "1.0.1", "time": "2024-01-01 10:00:00", "action": "更新"},
      {"version": "1.0.2", "time": "2024-01-02 15:30:00", "action": "更新"}
    ],
    "last_update": "2024-01-02 15:30:00"
  }
}
```

## 11. 架构优势

1. **模块解耦**: 通过代理层实现模块间通信，互不依赖
2. **分层清晰**: Routes → Services → DAL 职责明确
3. **前后端分离**: Vue SPA + Flask API，独立开发维护
4. **版本管理**: 内置版本控制，支持历史追溯
5. **跨平台**: path_utils 处理 Windows/Linux 路径差异
6. **可扩展**: 蓝图模式便于添加新模块
