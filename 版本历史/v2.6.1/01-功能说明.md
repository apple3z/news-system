## 🎯 新增功能

### 1. 综合排序功能

**支持 3 种排序方式**:

#### 1.1 时间排序 (默认)
```bash
# 按时间倒序（最新）
GET /api/news/search?sort_by=time&sort_order=desc

# 按时间正序（最早）
GET /api/news/search?sort_by=time&sort_order=asc
```

**应用场景**:
- 查看最新新闻
- 追踪事件发展时间线

#### 1.2 热度排序
```bash
# 按热度排序（最热）
GET /api/news/search?sort_by=hot_score&sort_order=desc

# 按热度排序（最冷）
GET /api/news/search?sort_by=hot_score&sort_order=asc
```

**应用场景**:
- 发现热门新闻
- 了解行业焦点

#### 1.3 相关度排序
```bash
# 按相关度排序（基于关键词）
GET /api/news/search?keyword=AI&sort_by=relevance
```

**排序规则**:
- 标题包含关键词：权重 3
- 摘要包含关键词：权重 2
- 内容包含关键词：权重 1
- 再按热度、时间排序

**应用场景**:
- 精确搜索相关新闻
- 关键词研究

## 🔧 技术实现

### 1. 排序逻辑

```python
# 构建排序子句
if sort_by == 'relevance' and keyword:
    # 相关度排序：根据关键词出现位置
    order_clause = '''
        CASE 
            WHEN title LIKE ? THEN 3
            WHEN summary LIKE ? THEN 2
            WHEN content LIKE ? THEN 1
            ELSE 0
        END DESC, hot_score DESC, time DESC
    '''
elif sort_by == 'hot_score':
    # 热度排序
    order_clause = f"hot_score {sort_order}, time DESC"
elif sort_by == 'time':
    # 时间排序（默认）
    order_clause = f"time {sort_order}, hot_score DESC"
else:
    # 默认排序
    order_clause = "time DESC, hot_score DESC"
```

### 2. 筛选逻辑

```python
# 新闻源筛选
if source:
    conditions.append("source = ?")
    params.append(source)

# 日期范围筛选
if date_from:
    conditions.append("time >= ?")
    params.append(date_from)

if date_to:
    conditions.append("time <= ?")
    params.append(date_to)
```

### 3. 新闻源列表

```python
@app.route('/api/news/sources')
def api_news_sources():
    """获取所有新闻源列表（v2.6.1 新增）"""
    conn = sqlite3.connect(NEWS_DB)
    c = conn.cursor()
    
    c.execute('''
        SELECT source, COUNT(*) as count
        FROM news
        WHERE source IS NOT NULL AND source != ''
        GROUP BY source
        ORDER BY count DESC
    ''')
    
    sources = [{'name': row[0], 'count': row[1]} for row in c.fetchall()]
    conn.close()
    
    return {'success': True, 'data': sources}
```

## ⚠️ 注意事项

### 1. 日期格式
- 必须使用 `YYYY-MM-DD` 格式
- 例如：`2026-03-10`

### 2. 排序优先级
- `sort_by=relevance` 仅在指定 `keyword` 时生效
- 否则自动降级为时间排序

### 3. 性能考虑
- 相关度排序需要额外 3 次 LIKE 查询
- 大数据量时建议使用时间/热度排序
